# KISA 블랙리스트에 등록됐는지 모르는 이유 — 그리고 확인 방법

> **채널:** 네이버 블로그 + 티스토리
> **목표 길이:** 1,200–1,500자
> **키워드:** KISA RBL, 이메일 차단 확인, 이메일 블랙리스트 한국
> **CTA:** GitHub 무료 진단 도구

---

## "이메일을 보냈는데 상대방이 못 받는다"고 할 때

이 상황을 경험해본 IT 담당자라면 공감할 겁니다. 발송 로그는 성공. 오류 메시지 없음. 그런데 수신자는 메일을 못 받았다고 합니다.

반송 이메일(NDR, Non-Delivery Report)도 오지 않습니다.

이 패턴이 반복된다면, 가장 먼저 의심해야 할 것이 **KISA RBL 등록 여부**입니다.

---

## KISA RBL이란?

KISA(한국인터넷진흥원)는 스팸 발신 IP를 실시간으로 차단 목록에 관리합니다. 이 목록이 **RBL(Real-time Blackhole List)**입니다.

국내 주요 메일 서비스 — 네이버 메일, 카카오(다음) 메일, KT, SK 브로드밴드 등 — 는 수신 메일 처리 시 KISA RBL을 조회합니다. 발신 IP가 목록에 있으면 메일을 **조용히 차단**합니다.

### 왜 모르는가 — 침묵하는 차단

스팸 필터가 발동하면 대부분의 메일 서버는 발신자에게 반송 메일을 보내지 않습니다. 반송 메일 자체가 스패머에게 "이 주소는 유효하다"는 정보를 주기 때문입니다.

결과적으로:
- 발송 서버: 전송 성공 기록
- 수신 서버: 조용히 폐기 또는 스팸 보관함 이동
- 발신자: 아무 알림 없음

이 구조 때문에 많은 기업이 KISA RBL에 등록된 채로 수개월을 보내기도 합니다.

---

## 누가 등록되는가

자신이 스팸을 보내지 않았더라도 RBL에 등록될 수 있습니다:

1. **공유 IP 피해:** 동일 서브넷의 다른 서버가 스팸을 대량 발송한 경우
2. **이전 IP의 오염:** 클라우드 인스턴스 IP를 재할당받았는데 이전 사용자가 스팸 발신
3. **계정 탈취:** 내부 사용자 계정이 해킹되어 스팸 발송에 악용된 경우
4. **대량 발송 오인:** 정상 마케팅 이메일이 수신자 신고로 RBL 임계값 초과

---

## 확인 방법

KISA RBL은 DNS 조회 방식으로 확인할 수 있습니다. 발신 IP를 역순으로 배열해 `rbl.kisa.or.kr` 앞에 붙입니다.

예를 들어 발신 IP가 `203.0.113.42`라면:

```bash
nslookup 42.113.0.203.rbl.kisa.or.kr
```

- 응답이 `NXDOMAIN`이면 → **등록되지 않음 (정상)**
- `127.0.0.2` 형태의 응답이 오면 → **블랙리스트 등록됨**

이 조회를 직접 수행하려면 발신 서버의 IP를 알아야 하고, 복수의 IP를 사용하는 경우 각각 확인해야 합니다.

---

## 해제 방법

KISA RBL에 등록된 경우 해제 신청은 KISA 스팸 대응 센터를 통해 합니다:

👉 **https://spam.kisa.or.kr**

해제 신청 시 스팸 발송 중단 및 보안 조치 완료를 증명하는 자료가 필요합니다. 처리 기간은 보통 2–5 영업일입니다.

---

## 자동 확인 도구

발신 IP 파악 → RBL 조회 → 국제 블랙리스트 조회 → SPF/DKIM/DMARC 확인까지 한 번에 처리하는 무료 CLI 도구입니다:

```bash
git clone https://github.com/pon00050/korean-email-deliverability
cd korean-email-deliverability
uv sync
uv run check.py yourdomain.co.kr
```

한국어 보고서 형태로 각 항목의 상태와 조치 방법을 안내합니다.

이메일 발송 문제가 있다면, KISA RBL 확인부터 시작하세요.
