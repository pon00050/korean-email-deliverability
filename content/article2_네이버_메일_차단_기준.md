# 네이버 메일이 우리 이메일을 차단하는 기준은?

> **채널:** 요즘IT 또는 네이버 블로그
> **목표 길이:** 1,500–2,000자
> **키워드:** 네이버 메일 스팸 차단, 네이버 메일 발송 안됨
> **CTA:** Article 1 링크 + GitHub 도구

---

## 들어가며 — 네이버 메일, 블랙박스의 내부

국내 이메일 수신 시장에서 네이버 메일(네이버 메일 + 한메일 포함 시 카카오 계열까지)은 약 70–80%를 차지합니다. 즉, 한국 기업이 이메일을 보낼 때 수신자의 대다수는 네이버 메일 사용자입니다.

그런데 네이버 메일의 스팸 필터 기준은 공식적으로 공개된 적이 없습니다. Gmail은 [Postmaster Tools](https://postmaster.google.com)를 통해 도메인 평판 및 스팸 비율을 발신자에게 제공합니다. 네이버는 이에 상응하는 도구가 없습니다.

이 글은 공개된 기술 표준과 실측 데이터를 바탕으로, 네이버 메일이 스팸 여부를 판정하는 4가지 핵심 신호를 설명합니다.

---

## 신호 1 — PTR(역방향 DNS): 가장 먼저 확인하는 신원 증명

메일 서버가 네이버로 연결을 시작하면, 네이버는 가장 먼저 발신 서버의 IP 주소로 역방향 DNS 조회(PTR lookup)를 실행합니다.

```
발신 IP: 203.0.113.42
PTR 조회 → mail.yourdomain.co.kr  ✅ 일치
PTR 조회 → (응답 없음)           ❌ PTR 없음
PTR 조회 → hostname.isp.net      ❌ 도메인 불일치
```

PTR이 없거나 발신 도메인과 일치하지 않으면, 네이버 메일은 해당 연결을 동적 IP(가정용 인터넷) 또는 무허가 서버로 추정합니다. 이 단계에서 이미 스팸 점수가 크게 올라갑니다.

PTR 설정은 메일 서버 IP를 할당한 ISP나 클라우드 공급자(AWS Route53, KT IDC 등)에서만 변경할 수 있습니다. 자체 DNS 설정으로는 불가능합니다.

---

## 신호 2 — SPF: "이 서버가 보낼 권한이 있는가"

SPF 레코드는 발신 도메인의 DNS에 등록된 허가 서버 목록입니다. 네이버 메일은 수신 시 SPF 확인(SPF check)을 수행합니다.

```
# DNS TXT 레코드 예시
yourdomain.co.kr TXT "v=spf1 include:_spf.google.com ip4:203.0.113.0/24 ~all"
```

`~all`(소프트페일)은 SPF에 없는 IP를 스팸으로 마크하지만 수신은 허용합니다.
`-all`(하드페일)은 SPF에 없는 IP의 메일을 거부합니다.

많은 기업이 클라우드 메일 서비스로 이전하면서 SPF를 업데이트하지 않아 자신의 발신 서버가 `~all` 실패 상태인지조차 모릅니다.

---

## 신호 3 — DKIM: 메일 내용의 무결성 보증

DKIM은 이메일 헤더와 본문의 일부를 해시하고 발신 도메인의 비공개 키로 서명합니다. 네이버 메일은 DNS에 등록된 공개 키로 이 서명을 검증합니다.

```
# DKIM 공개 키 DNS 레코드 예시
default._domainkey.yourdomain.co.kr TXT "v=DKIM1; k=rsa; p=MIGfMA0GCS..."
```

DKIM이 없는 메일:
- 전송 중 내용 변조 여부를 확인할 수 없음
- DMARC 정렬(alignment) 불가능 → DMARC 정책 작동 안 됨
- 네이버 스팸 필터에서 추가 감점

DKIM 서명이 없는 메일은 발신자 신원을 기술적으로 증명할 수 없는 메일입니다.

---

## 신호 4 — KISA 화이트도메인: 사전 등록 신뢰 신호

KISA 화이트도메인은 정상 발신 도메인으로 사전 신고한 도메인 목록입니다. 네이버를 포함한 국내 주요 메일 서비스는 이 목록을 스팸 필터 보정에 활용합니다.

등록된 도메인: 필터 임계값이 완화되어 오탐(정상 메일을 스팸으로 분류) 가능성 감소
미등록 도메인: 기본값 적용 — 다른 신호들의 점수로만 판정

화이트도메인 등록은 무료이며, 등록 후 수 일 내에 목록에 반영됩니다:
👉 https://spam.kisa.or.kr/white/sub2.do

---

## 4가지 신호 요약표

| 신호 | 확인 주체 | 설정 위치 | 없을 때 결과 |
|------|-----------|-----------|--------------|
| PTR | 네이버 메일 서버 | ISP/클라우드 공급자 | 스팸 점수 급증 |
| SPF | 네이버 메일 서버 | 발신 도메인 DNS | 스팸 마크 또는 거부 |
| DKIM | 네이버 메일 서버 | 발신 메일 서버 + DNS | 무결성 검증 불가 |
| KISA 화이트도메인 | 필터 보정 | KISA 신청 | 기본 필터 강도 적용 |

## 내 도메인은 어떤 상태인가요?

4가지 신호를 모두 수작업으로 점검하려면 별도 DNS 도구, 블랙리스트 조회 서비스, PTR 조회 등을 각각 사용해야 합니다.

아래 무료 CLI 도구는 이 모든 과정을 자동화해 한국어 보고서로 출력합니다:

```bash
git clone https://github.com/pon00050/korean-email-deliverability
cd korean-email-deliverability
uv sync
uv run check.py yourdomain.co.kr
```

→ 세금계산서 이메일이 스팸으로 가는 7가지 원인 전체 설명: [Article 1 링크]
