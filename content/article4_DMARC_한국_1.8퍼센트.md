# 한국 기업의 DMARC 설정률이 1.8%인 이유와 당장 해야 할 이유

> **채널:** 요즘IT 스타일 / 사고 리더십
> **목표 길이:** 1,200–1,800자
> **키워드:** DMARC 설정, 이메일 보안 한국, 도메인 인증
> **CTA:** GitHub 무료 진단 도구

---

## 숫자 하나 — 1.8%

Proofpoint의 2024 이메일 보안 현황 보고서에 따르면, 한국 기업의 DMARC 도입률은 **1.8%**입니다. APAC 전체 평균(약 12%)에 비해도 현저히 낮고, 글로벌 평균(약 55%)과 비교하면 사실상 미도입 수준입니다.

이 숫자가 의미하는 바는 단순합니다: 한국 기업 98.2%의 도메인은 누구든 쉽게 사칭할 수 있습니다.

---

## DMARC가 없으면 무슨 일이 생기나

### 1. 도메인 사칭(스푸핑)에 무방비

`noreply@yourcompany.co.kr`에서 온 것처럼 보이는 이메일을 아무나 보낼 수 있습니다. DMARC가 없으면 수신 서버에 "이 메일이 진짜 yourcompany.co.kr에서 온 것인지 확인하고, 가짜라면 어떻게 처리할지" 지시하는 방법이 없습니다.

세금계산서 피싱이 여기서 출발합니다. 공급업체 도메인을 사칭해 변조된 계좌번호의 세금계산서를 보내는 공격은 실제로 발생하고 있습니다.

### 2. ISP의 과잉 차단

DMARC가 없는 도메인은 스팸 필터 입장에서 "신원 불명"입니다. ISP는 불확실성에 보수적으로 대응합니다. 정상 이메일임에도 스팸으로 분류될 확률이 높아집니다.

### 3. DKIM만으로는 부족하다

DKIM 서명이 있어도 DMARC가 없으면 DKIM은 독립적으로 작동합니다. DMARC는 SPF와 DKIM을 정책으로 묶어 실제 도메인 인증 체계를 완성합니다.

---

## 왜 한국 기업들은 설정하지 않는가

이유는 복잡하지 않습니다.

**첫째, 모릅니다.** DMARC에 관한 한국어 설명이 거의 없습니다. Google 검색에서 상위 노출되는 DMARC 가이드는 대부분 영문이거나, 국내 내용이더라도 특정 메일 서비스(G Suite, Microsoft 365) 내 설정 방법에 국한됩니다.

**둘째, 당장 문제가 보이지 않습니다.** DMARC가 없어도 메일 발송 자체는 됩니다. 피해는 수신 측에서 조용히 발생하거나, 사칭 피해는 제3자에게 발생합니다. 설정 안 한 사람이 즉각적인 불이익을 받지 않습니다.

**셋째, 설정이 무섭습니다.** DNS를 잘못 건드리면 메일 서비스가 중단될 수 있다는 인식이 있습니다. 이 인식은 일부 사실이기도 합니다 — 잘못 설정한 DMARC는 정상 메일을 거부할 수 있습니다.

## 당장 해야 하는 이유

### Google/Yahoo 발신자 요건 (2024년 시행)

2024년 2월부터 Google과 Yahoo는 하루 5,000건 이상 이메일을 발송하는 도메인에 SPF, DKIM, DMARC 설정을 **의무**로 요구하고 있습니다. 미설정 시 메일이 거부됩니다.

세금계산서를 대량 발송하는 Tax SaaS 도메인은 이 기준에 해당할 가능성이 높습니다.

### 단계적 적용으로 리스크 없이 시작 가능

DMARC는 `p=none`(모니터링 모드)으로 시작할 수 있습니다. 이 설정은 메일 처리에 영향을 주지 않고, SPF/DKIM 통과 현황 리포트만 수집합니다.

```dns
_dmarc.yourdomain.co.kr TXT "v=DMARC1; p=none; rua=mailto:dmarc-reports@yourdomain.co.kr"
```

리포트를 2–4주 수집해 발신 패턴을 파악한 후, `p=quarantine` → `p=reject` 순으로 강화합니다.

## 내 도메인의 DMARC 상태 확인

```bash
# 명령줄에서 직접 확인
nslookup -type=TXT _dmarc.yourdomain.co.kr

# 아무 응답도 없으면 DMARC 미설정
```

또는 아래 무료 진단 도구로 SPF, DKIM, DMARC, PTR, RBL 전체를 한 번에 확인하세요:

```bash
git clone https://github.com/pon00050/korean-email-deliverability
cd korean-email-deliverability
uv sync
uv run check.py yourdomain.co.kr
```

1.8% 중 하나가 될지, 98.2%에서 벗어날지 — 선택은 DNS 레코드 한 줄입니다.
