# 한국 핀테크/금융 — 이메일 Deliverability 컨설팅 잠재 클라이언트 리서치
**작성일:** 2026-02-28
**목적:** 한국 이메일 deliverability 컨설팅 사업의 유료 클라이언트 후보 식별

---

## 개요: 왜 핀테크가 최우선 세그먼트인가

이메일 deliverability는 대부분의 기업에서 "있으면 좋은" 수준이지만, 핀테크·금융에서는 **직접적 매출 손실**로 이어진다.

| 이메일 유형 | 실패 결과 | 금전적 영향 |
|---|---|---|
| 이메일 OTP/인증 | 로그인 실패, 회원가입 이탈 | 고객 획득 비용 직접 소멸 |
| 거래 확인 이메일 | 거래 완료 불신, CS 폭주 | 운영비 + 이탈률 |
| 해외 송금 확인 | 규정 준수 미이행 | 금감원 리스크 |
| 보험료 납입 고지 | 납입 누락 → 실효 → 민원 | 금감원 민원 + 손해배상 |
| 증권 계좌 알림 | 주문 확인 미수신 | 고객 클레임 |
| 이상 거래 경보 | 사기 피해 확대 | 배상 책임 |

**한국 특수 문제:** DMARC 채택률 1.8% (APAC 최하위). 금융 기업조차 인증 미설정 상태로 발송하는 경우가 다수. Gmail/Yahoo의 2024년 대량발송 강화 이후 핀테크 수신율 저하 가능성 높음.

## 담당 부서별 접근 전략

### CTO/개발팀 접근 (소형 스타트업)
- **대상:** 모인, 보맵, 렌딧
- **메시지:** "이메일 OTP 전송 실패율 X% 감소 = 회원가입 전환율 직결"
- **진입:** LinkedIn 직접 접촉 / 개발자 커뮤니티 (OKKY, 스타트업 슬랙)
- **예상 계약:** 프로젝트성 감사 (100–300만 원) + 기초 구현 (300–500만 원)

### 보안팀 접근 (가상자산, 인터넷은행)
- **대상:** 업비트, 빗썸, 카카오뱅크
- **메시지:** "DMARC p=reject 미설정 = 브랜드 도용 피싱 이메일 차단 불가. 해킹 이후 법적 책임 리스크"
- **진입:** 보안 컨퍼런스 (ISEC, 금융보안원 세미나)
- **예상 계약:** 보안 감사 리테이너 (월 200–500만 원)

### 컴플라이언스팀 접근 (보험, 증권, 로보어드바이저)
- **대상:** 카카오페이보험, 토스증권, 파운트
- **메시지:** "법적 고지 이메일이 스팸 분류 → 금감원 민원 + 금융소비자보호법 위반 리스크"
- **진입:** 법무법인 연계 / 준법감시 네트워크
- **예상 계약:** 규정 준수 감사 + 연간 deliverability 인증

### 플랫폼 인프라팀 접근 (대형 핀테크)
- **대상:** 토스 본체, 카카오페이
- **메시지:** "글로벌 확장 시 각국 ISP 규정 상이 → 해외 사용자 이메일 도달률 보증 불가"
- **진입:** 기술 블로그/컨퍼런스 발표 → 인바운드
- **예상 계약:** 프로젝트 + 월 리테이너 (500만 원+)

## 전략적 시사점

1. **즉시 접근 가능한 1순위:** 센트비, 모인, 토스페이먼츠, 빗썸 — 고통이 명확하고 의사결정이 빠름. 첫 유료 계약 목표 대상.

2. **M&A 트리거 활용:** 카카오페이의 SSG Pay / Smile Pay 인수 (2025, ₩5,000억)는 새로운 발송 도메인 통합 이슈를 즉시 발생시킴. 이 타이밍에 "M&A 이후 이메일 인프라 통합 감사" 포지셔닝.

3. **규제 레버리지:** 가상자산이용자보호법 (2024 시행)이 VASP의 고객 통지 의무를 강화함. 업비트·빗썸에 대해 "법적 고지 deliverability 보장" 서비스로 접근.

4. **Korea-specific 차별화:** 어떤 글로벌 deliverability 도구도 Naver·Kakao 인박스 시드 데이터를 보유하지 않음. 이것이 핵심 경쟁 우위 — 한국 핀테크는 사용자의 상당 비율이 Naver 메일 사용자.

5. **레퍼런스 구축 경로:** 소형 스타트업(모인, 렌딧) → 레퍼런스 확보 → 중형(센트비, 빗썸) → 대형(토스, 카카오페이) 순으로 단계적 확장.
