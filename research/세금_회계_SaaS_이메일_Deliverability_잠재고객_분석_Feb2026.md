# 한국 세금·회계 SaaS — 이메일 Deliverability 잠재 고객 분석
**작성일:** 2026-02-28
**목적:** 이메일 Deliverability 컨설팅 서비스의 유료 고객 후보군 식별 및 접근 전략

---

## 핵심 전제: 왜 세금·회계 SaaS인가

전자세금계산서(e-invoice) 이메일은 **법적 의무 통지**다. 전송 실패 = 가산세 발생 + 거래 분쟁 가능성. 이 카테고리의 이메일은 마케팅 이메일과 달리 "안 와도 그만"이 아니다. 도달 실패의 재무적 결과가 수치화되어 있기 때문에 deliverability 투자의 ROI 계산이 명확하다. 이것이 일반 마케팅 SaaS보다 훨씬 강력한 영업 논거다.

---

## 법적 배경: 전자세금계산서 이메일 불이행 가산세

| 위반 유형 | 가산세율 | 근거 |
|---|---|---|
| 전자세금계산서 미전송 (발급일 다음날까지) | 공급가액의 **0.3%** | 부가가치세법 §60 |
| 이메일 전송 오류로 반송 후 기한 내 미재발급 | 공급가액의 **0.5%** | 부가가치세법 §60 |
| 발급 자체 지연 (의무발행 사업자) | 공급가액의 **1~2%** | 부가가치세법 §60 |

**핵심 판단:** 이메일이 스팸함으로 들어가거나 반송되면 법적으로 "미발급"으로 간주될 수 있다. 플랫폼 운영사는 고객(세무사, 기업)으로부터 클레임을 받는다. Deliverability 실패는 플랫폼의 신뢰도 문제이자 고객 이탈 원인이다.

**확인된 실제 문제 사례:**
- WEHAGO(더존 위하고)의 발신 주소 `webmaster@bill36524.com`이 한메일(Daum)에서 "존재하지 않는 메일주소"로 반환되는 사례 → 더존 고객센터 공식 FAQ로 등재
- Gmail에서 계산서 발행 관련 외부 이메일 수신 불가 문제 → Google 커뮤니티 보고 다수
- 국세청 사칭 피싱 이메일이 정상 전자세금계산서 도메인과 혼동되어 정상 메일까지 차단되는 현상

## 잠재 고객 명단 (8–12개사)

### Tier 1 — 최우선 접근 (이메일 발송 규모 + 법적 의무 직결)

#### 1. 더존비즈온 — 위하고(WEHAGO) / 이큐브(iCUBE)

| 항목 | 내용 |
|---|---|
| **서비스** | 위하고: 전자세금계산서 통합 플랫폼 (구 Bill36524 흡수). iCUBE: 중견·대기업 ERP |
| **이메일 발송 규모** | 국내 최대. 연간 수천만 건 전자세금계산서 이메일 발송 추정 (ERP 시장점유율 1위) |
| **발신 도메인** | `bill36524.com`, `wehago.com`, `douzone.com` |
| **확인된 문제** | `webmaster@bill36524.com` 한메일 차단 사례 공식 FAQ화 — 즉, 문제를 인지하지만 근본 해결 못함 |
| **법적 리스크** | 수십만 법인 고객의 전자세금계산서 이메일이 스팸 처리 → 가산세 클레임 |
| **추가 맥락** | EQT(스웨덴 PE) 1.3조 인수(2025.11) 후 글로벌 스탠더드 요구 ↑. SPF/DKIM/DMARC 미정비 = 글로벌 기준 미달 |
| **deliverability 노출** | 전자세금계산서 알림, 거래처 공지, 위하고 회원 이메일, 급여명세서 연동 |

**접근 담당자:** 위하고 서비스 기획 담당 (Product Manager) > CTO 산하 인프라팀. B2B 계약이므로 기획/운영 팀장급이 구매 결정권자.

#### 2. 한국세무사회전산법인 — 플랫폼세무사회 / 국민의세무사 / AI세무사

| 항목 | 내용 |
|---|---|
| **서비스** | 플랫폼세무사회(세무사 전용 통합 업무 플랫폼), 국민의세무사(소비자 앱), AI세무사 |
| **이메일 발송 유형** | 세무사 회원 공지, 의무 교육 알림, 고객 신고 완료 통지, 세무사회 뉴스레터 |
| **발신 도메인** | `hangiltis.com`, `semuportal.com`, `ctaplatform.com` |
| **법적 리스크** | 세무사 회원(1만 3천명+)에게 발송하는 공지 미도달 → 의무 연수 누락, 법령 개정 공지 미수신 |
| **전략적 의의** | **나의 기획자 취업 목표 기업.** 내부 기획자 = 이메일 인프라 의사결정 직접 관여 가능 |
| **특이사항** | 국민의세무사 앱 출시(2025) 후 회원 onboarding 이메일, 앱 알림 병행 → transactional 이메일 급증 예상 |

**접근 담당자 (외부 컨설턴트로서):** 대표이사 또는 IT개발 총괄 (소규모 법인이므로 CTO = 개발팀장 수준). 기획자 채용 후 내부에서 직접 제안 가능.

| **더존 위하고** | 전자세금계산서 | 가산세 0.3–1% | 수천만 건/년 | Implementation + 인프라 audit + retainer |

| **국민의세무사 앱 기획** | 앱 onboarding 이메일, 세무 상담 결과 통지 이메일 설계 |
| **AI세무사 기획** | AI 분석 결과 리포트 이메일 발송 설계 |
| **전산법인 회원 관리** | 세무사회 전체 연락망 → 이메일 발송 인프라 관리 |
| **외부 협력사 관계** | 더존·바로빌·세무사랑 등 연동 시스템과의 협력 → 업계 네트워크 |

### 내부 기획자 → 외부 컨설턴트 전환 경로

```
전산법인 기획자 (1–2년)
    ↓
→ 플랫폼 이메일 인프라 실태 파악
→ 세무사 회원들의 실제 이메일 문제 수집 (1.3만명 직접 접점)
→ 더존·바로빌 등 업계 벤더 담당자 관계 형성
→ "세무사 업계 이메일 deliverability 전문가" 포지션 확보
    ↓
컨설팅 독립 or 세무 SaaS 벤더 B2B 영업
```

**핵심 인사이트:** 전산법인 기획자 포지션은 업계 이메일 문제의 **수요측(세무사)과 공급측(플랫폼)을 동시에 볼 수 있는 유일한 자리**다. 외부에서 접근할 때 가장 약한 부분(실제 문제가 얼마나 심각한지 내부 데이터)을 채워준다.

## 서비스 패키지 설계

### Phase 1: 진단 (Implementation)

**"세금·회계 SaaS 이메일 인프라 Audit"** (1회성, 300–500만원)

- SPF / DKIM / DMARC 설정 진단
- 발신 도메인 reputation 조회 (MXToolbox, Google Postmaster, 국내 블랙리스트)
- Naver Mail / Daum(Kakao) 필터링 테스트 (seed account panel)
- KISA 화이트도메인 등록 상태 확인
- 발송 볼륨 vs. IP warm-up 상태 진단
- 보고서 + 개선 로드맵 제공

**ROI 논거:** 전자세금계산서 1건 미도달 시 가산세 = 공급가액 0.3–0.5%. 중소 플랫폼도 일 1만 건 발송 시 가산세 노출 금액이 월 수백만원 이상.

---

### Phase 2: 구축 (Implementation)

**"한국형 Deliverability 인프라 구축"** (프로젝트, 500–1500만원)

- DMARC 단계적 적용 (p=none → quarantine → reject)
- 발신 IP 분리 (transactional vs. marketing)
- Naver / Daum 수신 허용 도메인 등록 지원
- KISA 화이트도메인 신청 대행
- Bounce 처리 자동화 설계
- 반송 이메일 재발송 프로세스 (법적 기한 내)

---

### Phase 3: 유지 (Retainer)

**"한국 Inbox 모니터링 월정 서비스"** (월 100–300만원)

- Naver / Daum / Gmail seed account panel 주간 도달률 보고
- Blacklist 모니터링 + 즉시 알림
- 발신 도메인 reputation 추이
- 국내 필터링 정책 변경 모니터링 (Naver Mail 업데이트, KISA 정책)
- 분기 전략 리뷰
