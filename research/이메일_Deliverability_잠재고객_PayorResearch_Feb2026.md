# 한국 이메일 Deliverability — 잠재 고객(Payor) 리서치
**작성일:** 2026년 2월 28일
**목적:** 이메일 deliverability 컨설팅 서비스의 실제 지불 주체(payor)를 세그먼트별로 파악하고, 긴급도 × 예산 × 접근성 × 서비스 적합도 기준으로 우선순위화

> **배경 전제:** 한국은 DMARC 도입률 1.8% (APAC 최하위). 한국어 deliverability 전문가 없음. Naver Mail 필터링 로직 완전 비공개. 이 공백이 본 컨설팅의 시장 기회.

### 접근 전략

- **스티비 파트너 등록이 핵심 레버:** 한국 SaaS 이메일 마케터 중 스티비 사용자가 다수. 스티비 공식 파트너/블로그 기고자 포지셔닝 → 채널톡, MyRealTrip 등에 warm intro 경로 생성. 이 하나의 액션이 15개 중 7–8개 접근 온도를 바꿈.
- **즉시 접근 3개 (3월):** Relate (CRM = 이메일 자체가 제품), Specter (레퍼런스 이메일 bounce = 즉각적 제품 실패), Typed (글로벌 론칭 직후)
- **LinkedIn 직접 메일 pitch:** "레퍼런스 체크 이메일이 Outlook 스팸으로 빠지면 어떻게 되나요?"

**현재 고객 에스컬레이션 데드엔드:**

### 그룹웨어 vs 마케팅/트랜잭셔널 이메일 분리 구조

한국 모든 주요 그룹웨어(하이웍스, 메일플러그, 더존 위하고, 영림원, 카카오워크, 네이버웍스)는 **내부 커뮤니케이션**에 최적화. 대량 마케팅/트랜잭셔널 이메일은 **별도 ESP 필요**. 이 분리 지점이 SPF/DKIM 불일치 발생 지점 → 그룹웨어 사용 기업 전체가 잠재 클라이언트.

### 세그먼트별 우선순위

| 순위 | 세그먼트 | 긴급도 이유 |
|---|---|---|
| **1** | 이커머스/D2C | 이메일 bounce = 직접 매출 손실; 마케터가 문제 인식함 |
| **2** | B2B SaaS | 제품 이메일이 핵심 UX; CTO가 기술 이해 |
| **3** | HR/채용 플랫폼 | 중요 알림 미도달 = 브랜드 피해; 민감도 높음 |
| **4** | 핀테크/금융 | 규제 + 고객 신뢰 연동; 예산 풍부 |
| **5** | 제조 중견기업 | 규모 크나 인식 낮음; 설득 시간 필요 |

### Barracuda Networks 한국 현황

시큐와이드(주)가 공식 한국 수입원 (2011년~). 취급: 이메일 보안 게이트웨이(BSG), WAF, 스팸/바이러스 방화벽. 도입 고객: 데이터 내부화 선호 중견~대기업, 금융, 공공기관. **단, 안랩(AhnLab)이 국내 이메일 보안 1위 → Barracuda 점유율은 제한적.** 역진입 프레이밍 가능: "귀사 고객사의 Barracuda 필터가 귀사 마케팅 이메일을 차단하고 있을 수 있습니다."

NHN Cloud가 공공기관 클라우드 47% 점유 → 공공 이메일 발송 인프라 상당 비중이 NHN Cloud Email 기반. 한국 공공기관 DMARC 도입률 극히 저조(추정 10% 미만; 미국 연방기관 87.2% 대비). KISA가 도입률 향상을 정책 과제로 인식 → KISA 협력 레퍼런스 = 공공기관 진입 명함. 단, 조달청 나라장터/디지털서비스몰 등록이 선행 조건 (18개월+ 소요) → 단기 전략 아님.

### 이메일 Deliverability의 기업 내 소유권 공백

**핵심 인사이트:** 한국 기업에서 이메일 deliverability를 "소유"하는 부서가 **없다.** 마케팅팀은 수치를 보지만 기술적 원인 모름; IT팀은 DNS 권한 있지만 마케팅 이메일 비관심; 보안팀은 수신 보안에만 집중. 이 소유권 공백 = 외부 전문 컨설턴트의 가치 명제. **진입 순서:** 마케팅팀(챔피언) → IT팀(실행 파트너) → 경영진(예산 승인).
